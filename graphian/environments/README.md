# environments ── 環境・身体の空き枠(§1 / §7.1)

身体と環境の応答ループを提供する層。1D → 2D → 3D → 抽象空間と「増えていくもの」であり、
core に触れずに python module として追加できる。

## 現状(初版)

- `base.py` : `Environment`(抽象基底=契約の一次資料)。単体個体モデル(集団は不要=決定2)。
- ダミー兼参照実装 `phototaxis_1d.py`(1 次元走光性, §7.1)は本段階では未実装。

## 新版を作る者へ(§10)

`Environment` を継承し、感覚接続点・運動接続点を `connection_layout` で宣言する。`reset` で
`buffer.declare_layout()` を呼んで形を伝え、`step` で「運動を読む → 世界更新 → 生死判定 →
感覚を書く」を実装する。生死判定は環境が所有し(§3 ③)、適応度は連続量にする(§7.1)。
反射系(前処理)は各環境モジュール内の拡張ポイントとしてのみ確保する(core には組み込まない)。
