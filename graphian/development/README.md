# development ── 発達ルールの空き枠(§1 / §8)

ネットワークが「どう発達するか」を定める層。**意図的に「空き枠」として隔離**されており、
core には一切触れずに中身を差し替えられる。

## 現状(初版)

- `base.py` : `DevelopmentRule`(抽象基底=契約の一次資料)、`DevelopmentContext`、`Budget`。
  **数学は含まない。** 契約はライフサイクル(`initialize` / `step`)のみを固定する。
- ダミー発達ルール(`dummy.py`, §7.2)は本段階では未実装。

## ここに数学を実装してはならない(初版指示)

報酬系の計算・確率分布・活動依存の発達則などを先回りで考案・実装しないこと。本物の数学は
後続の「数学設計セッション」で、`DevelopmentRule` 契約を満たす具象として追加する(§6 / §8)。

## 系統との対応(決定1)

情報処理基盤の構造(値の型・統合・活性化・伝播)と発達のさせ方は **系統(§9.5)ごとに異なり、
§8 の数学と一対一対応する**。したがって具象 `Network` と具象 `DevelopmentRule` はこの層
(または系統固有モジュール)に置かれ、core には固定しない。
